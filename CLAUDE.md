## Commands

```sh
# Backend (Django, port 8000)
python manage.py runserver
python manage.py test                      # all 83 tests
python manage.py test gym.tests.YourTest   # single test class
python manage.py seed_workouts             # seed sample workout history

# Frontend (React + Vite, port 5173, proxies /api -> :8000) — separate terminal
cd frontend && npm run dev
cd frontend && npm run build               # production build to frontend/dist/
```

- Python 3.12, Django 5.2, DRF, SQLite. **No requirements.txt** — Django + djangorestframework + django-cors-headers assumed installed globally/conda.
- Node 22, React 18, Vite 6, React Router 6. No frontend lint/test scripts exist.
- Demo user: `user` / `1234` (seeded by migration `0002_seed_data`).

## Architecture

Two-tier app: Django REST backend + React SPA frontend, session-cookie auth.

**Backend** (`gym/` app, `helloworld/` project config):
- `gym/api_views.py` — all REST endpoints, function-based with `@api_view` (no ViewSets). **All new backend work goes here.**
- `gym/serializers.py` — DRF serializers.
- `gym/views.py` + `gym/templates/gym/` — legacy template views, kept only because tests cover them. Don't build new features on them.
- `gym/models.py` — Exercise, DayPreset, DayPresetExercise, WorkoutDay, WorkoutExercise, WorkoutSet.
- Unauthenticated API calls return 403 (session auth).

**Frontend** (`frontend/src/`):
- Pages: Login, Dashboard, Exercises, DayCreate, DayDetail, PresetForm, PresetDetail.
- Components: Navbar, Sidebar, MobileTopBar, FlashMessages.
- `api.js` — single fetch-based API client; all requests go through it as JSON.
- `index.css` — all styles; dark/light theme via `data-theme` attribute + `localStorage`.

API endpoints are all under `/api/` — see the table in AGENTS.md.

## Gotchas

- Seed migration creates 36 exercises; names like "Bench Press" collide with test fixtures — use `ZZ`-prefixed exercise names in tests.
- Tests cover models + legacy template views + integration flows; **API views are untested**.
- `aggregate(...)["order__max"] or -1` pattern: order 0 is falsy — use explicit `is None` checks when computing next order/sequence numbers.
- Migrations were generated by Django 6.0.7 but run on 5.2; recreate them if upgrading Django.
- `admin.py` is an empty stub — no models registered.
