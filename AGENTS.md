# Gym Logger — AGENTS.md

## Quick start
```sh
# Backend (Django)
python manage.py runserver        # dev server on :8000
python manage.py test              # run all tests (Django TestCase)

# Frontend (React + Vite) — in another terminal:
cd frontend && npm run dev         # dev server on :5173, proxies /api -> :8000
```

- Python 3.12, Django 5.2, SQLite. No virtual env required (conda/global install).
- Node.js 22 + npm, React 18, Vite 6, React Router 6.
- Demo user: `user` / `1234` (seeded by migration `0002_seed_data`)

## Architecture

**Backend (Django)** — REST API at `/api/*` using DRF with session auth.
- `gym/api_views.py` — all API views (`@api_view`, function-based)
- `gym/serializers.py` — DRF serializers for all models
- Template-based views in `gym/views.py` still exist for legacy support (tested).
- No requirement to add new template views; all new work goes in `api_views.py`.

**Frontend (React)** — SPA in `frontend/` served by Vite dev server.
- Vite proxies `/api/*` to Django on port 8000.
- Pages in `frontend/src/pages/`, shared components in `frontend/src/components/`.
- `frontend/src/api.js` — fetch wrapper for all API calls.
- CSS in `frontend/src/index.css` (mirrors the old template styles exactly).


## Important quirks

- **No requirements.txt** — Django + DRF + django-cors-headers must be installed.
- **`DEFAULT_AUTO_FIELD`** is now set in settings (warning fixed).
- **No models registered in admin** — `admin.py` is an empty stub.
- **Duplicate import** in `views.py:10` — `from django.db import models, transaction` duplicates line 4's `from django.db import transaction`. Could consolidate.
- **`0 or -1` bug fixed** in `day_add_exercise` — `max_order = aggregate(...)["order__max"] or -1` treats order `0` as falsy, giving `-1`. Changed to explicit `is None` check. Same pattern in `set_add` (`or 0`) is safe because set_numbers start at 1.
- **Auth**: session-based. Login sets a Django session cookie. Unauthenticated API calls return 403.
- **Migrations** were generated by Django 6.0.7 but run on 5.2 — no known issues, but recreate them if upgrading Django.
- **Tests** (83 tests) cover models + all template views + integration flows. API views are not yet tested.
- **Seed data** migration creates 36 exercises + demo user. Exercise names like "Bench Press" conflict with test fixtures — use `ZZ`-prefixed names in tests.

## Conventions

- API views use `@api_view` decorator + DRF `Response`. No ViewSets.
- Serializers are in `gym/serializers.py`.
- Frontend pages use `react-router-dom` v6 with `useNavigate`, `useParams`, `Link`.
- CSS variables match the old template theme (dark/light via `data-theme` + `localStorage`).
- Form data is sent as JSON via fetch. No Django forms on the frontend.
